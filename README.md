# 原神圣经群聊 Skill

一个面向 AstrBot / 群聊 Bot 的反串型梗库 Skill。它会在群聊里根据触发词调用“原神圣经”式话术，用夸张、抽象、爹味、护主、赛博宗教和宏大叙事的方式接梗。

> 这不是官方资料库，也不是认真吵架工具。它的定位是群聊节目效果：让 Bot 在合适场景里像“原神抽象护教士”一样发癫。

## 功能特点

- **19 个梗档案**：覆盖跳过剧情、尖锐爆鸣、648、福利、海灯节、文化输出、核聚变、建模、退坑等常见群聊触发点。
- **触发词 + 语义触发**：既可以靠关键词触发，也可以根据上下文识别“吐槽/对线/玩梗”语境。
- **短打优先**：默认适合群聊 1～4 句话回复，避免每次都刷屏小作文。
- **知识库友好**：Skill 负责梗和语气，原神知识库负责角色、剧情、设定、任务、玩法等事实内容。
- **安全边界**：保留反串味，但不鼓励真实人身攻击、开盒、人肉、现实威胁或网暴。

## 快速安装

### 方法一：直接上传 ZIP

1. 下载本仓库的 ZIP 包，或在 GitHub 页面点击 `Code` → `Download ZIP`。
2. 确认 ZIP 解压后结构类似：

   ```text
   genshin-bible-groupchat-skill/
   └── SKILL.md
   ```

3. 进入 AstrBot 的 Skill / 插件管理页面。
4. 上传 ZIP 包。
5. 如果提示校验成功，启用该 Skill。
6. 在群聊中测试：

   ```text
   原神剧情太长了，能不能跳过？
   ```

   预期效果类似：

   ```text
   跳过剧情就是跳过人生。你以为你想省的是十分钟，其实你正在亲手删掉自己灵魂里本该发光的部分。
   ```

### 方法二：手动放入 Skill 目录

适合服务器部署或面板上传失败时使用。

1. 将 `genshin-bible-groupchat-skill` 整个文件夹复制到 AstrBot 的 Skill / 插件目录。
2. 确保文件夹内存在 `SKILL.md`。
3. 重启 AstrBot 或在管理面板中重新加载 Skill。
4. 如果校验失败，优先检查：
   - ZIP 根目录是不是多套了一层文件夹；
   - `SKILL.md` 文件名是否大写；
   - `SKILL.md` frontmatter 是否保留在文件开头；
   - 文件编码是否为 UTF-8。

## 推荐人格提示词

建议把 `docs/persona-prompt.md` 里的内容放到 Bot 的人格 / 系统提示词中。核心思路是：

- 遇到梗、吐槽、对线：优先调用 Skill。
- 遇到角色、剧情、设定、任务：优先检索知识库。
- 默认短打，不要每次写长篇。
- 禁止动作描写、心理描写和旁白叙述。

## 和知识库搭配使用

这个 Skill 不建议承担事实检索。更稳的做法是：

```text
知识库：存原神正式设定、角色、国家、任务、组织、玩法、社区术语。
Skill：负责触发梗、生成语气、抽象升华和群聊节目效果。
人格提示词：负责规定“什么时候查知识库，什么时候调用 Skill”。
```

当群友问：

```text
芙宁娜是谁？
```

Bot 应该先查知识库，回答角色背景。

当群友说：

```text
芙宁娜也没那么值钱吧？
```

Bot 才适合调用 Skill 里的“金融神童 / 560 亿 IP 价值”梗。

详细配置见：[`docs/knowledge-base-guide.md`](docs/knowledge-base-guide.md)。

## 调整回复强度

建议在 Bot 人格中设置默认强度：

- **20%**：轻度阴阳怪气，只接一句。
- **40%**：普通群聊玩梗。
- **60%**：标准原神圣经人格。
- **80%**：群友主动开团时使用。
- **100%**：群友明确要求“来段圣经 / 狠狠发癫”时才使用。

不要让 Bot 每条都用 100% 小作文，否则群聊体验会很吵。

## 安全边界

本项目是反串梗库，但仍建议保留以下边界：

- 不输出真实人身攻击。
- 不进行开盒、人肉、现实威胁、线下报复或网暴引导。
- 不攻击现实身份、地域、性别、疾病、家庭等个人属性。
- 不冒充米哈游、HoYoverse、原神官方或任何社区管理员。
- 不把 Skill 中的夸张说法当成事实。
- 群友明显不舒服时，降低强度或切回正常回答。
